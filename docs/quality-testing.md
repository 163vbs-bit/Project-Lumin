# QA и отчетность

Для демонстрации можно запускать автоматический QA-набор:

```bash
python qa-tests/run_quality_suite.py --users 10
```

Параметр `--users` принимает значения от 1 до 100. Для проверки нескольких уровней нагрузки используется `--load-steps`:

```bash
python qa-tests/run_quality_suite.py --users 10 --load-steps 1,10,50,100
```

Каждый бот:

- регистрирует уникальный аккаунт;
- входит в систему;
- открывает профиль, аналитику, категории, список тестов и рейтинг;
- получает тест;
- отправляет попытку.

После запуска создается папка `reports/qa_run_...`.

## Структура отчета

- `README.md` — краткое описание запуска.
- `summary.csv` — общая сводка по всем разделам.
- `report.html` — удобная HTML-сводка для просмотра в браузере.
- `verification/verification_checks.csv` — проверка доступности корня приложения, healthcheck, OpenAPI, категорий и тестов.
- `validation/validation_accounts.csv` — валидные и невалидные регистрации, вход с правильным и неправильным паролем.
- `usability/usability_checks.csv` — доступность frontend-маршрутов и проверка качества тестовых данных.
- `load/load_matrix.csv` — скорость работы и память при выбранных уровнях нагрузки.
- `load/load_accounts.csv` — каждый бот в отдельной строке.
- `load/load_steps.csv` — каждый шаг каждого бота.
- `load/performance_summary.csv` — среднее время, P50, P95, throughput.
- `load/memory_samples.csv` — замеры `docker stats`.
- `load/memory_summary.csv` — среднее и максимальное потребление памяти по контейнерам.

Если задано несколько уровней нагрузки, например `--load-steps 1,10,50,100`, подробные таблицы лежат в подпапках `load/1_users`, `load/10_users`, `load/50_users`, `load/100_users`, а общая сравнительная таблица остается в `load/load_matrix.csv`.

## Что считается проверкой

- Верификация: система доступна, API описан через OpenAPI, есть категории и тесты, первый тест содержит вопросы.
- Валидация: регистрация принимает корректные данные студента и преподавателя, отклоняет некорректные email, короткий пароль, неверную роль, дубли логина/почты, проверяет вход и доступ преподавателя к teacher-only API.
- Юзабилити: основные маршруты открываются, тестовые данные не содержат повторов, пустых ответов, отсутствующего кода и битой кодировки.
- Нагрузка: реальные HTTP-запросы создают аккаунты, выполняют вход, читают данные, проходят тест и фиксируют время каждого шага.
